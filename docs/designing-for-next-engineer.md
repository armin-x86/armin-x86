# Cross-Functional Teams and Shared Understanding

Infrastructure exists to support software.

If developers cannot understand the environment they deploy to, collaboration breaks down.

Likewise, infrastructure engineers must understand enough about applications to design useful platforms.

Healthy teams therefore require **cross-functional understanding**.

This does not mean everyone must be an expert in everything.

It means engineers should have enough shared knowledge to collaborate effectively.

A useful analogy is a kitchen.

Different people may specialize in:

- preparing ingredients
- cooking dishes
- plating food

But if one person becomes overwhelmed, others should be capable of stepping in and helping.

In engineering teams, this translates to:

- developers understanding deployment patterns
- platform engineers understanding application needs
- shared knowledge of how systems operate

Infrastructure code should support that collaboration rather than obscure it.
